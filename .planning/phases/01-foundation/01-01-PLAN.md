## Trust Boundaries

| Boundary                                | Description                                                                                      |
| --------------------------------------- | ------------------------------------------------------------------------------------------------ |
| Local dev machine → HA host (rsync/ssh) | Source files copied to a remote HA host over SSH; deploy command runs `ha core restart` remotely |
| Repo → HA integration loader            | manifest.json controls how HA loads the integration                                              |

## STRIDE Threat Register

| Threat ID | Category               | Component                                                                       | Disposition | Mitigation Plan                                                                                                                                                                                     |
| --------- | ---------------------- | ------------------------------------------------------------------------------- | ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| T-01-01   | Tampering              | Makefile deploy (`rsync --delete` to /config/custom_components/climate_manager) | mitigate    | rsync target path is scoped to the integration subdirectory only (HA_COMPONENT_DIR = /config/custom_components/climate_manager), never the parent /config — `--delete` cannot wipe unrelated config |
| T-01-02   | Information Disclosure | HA_USER / HA_HOST in Makefile                                                   | accept      | Defaults are non-secret placeholders (root@homeassistant.local) overridable via env vars; no credentials are stored in the repo (SSH key auth is the user's responsibility)                         |
| T-01-03   | Spoofing               | SSH connection to HA host                                                       | transfer    | SSH host key verification and key-based auth are delegated to the user's SSH configuration; out of integration scope per RESEARCH Security Domain                                                   |
| T-01-04   | Denial of Service      | manifest.json malformed → integration fails to load                             | mitigate    | Task 1 verification asserts manifest.json parses as valid JSON with required fields before the plan completes                                                                                       |

</threat_model>
