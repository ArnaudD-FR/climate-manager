| Integration → climate domain services | TRV service calls cross from this integration into HA's climate platform |
| HA `.storage/` filesystem ↔ Store | Persisted config serialized to disk via the Store helper |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Tampering | storage.py Store data on disk | accept | HA owns `.storage/` file permissions; no PII or secrets in the schema (room/person config only); RESEARCH Security Domain confirms V6 not applicable for Phase 1 |
| T-01-06 | Denial of Service | Blocking I/O in async storage path freezes HA event loop | mitigate | storage.py uses homeassistant.helpers.storage.Store exclusively; acceptance criteria assert no open()/json.load/json.dump in storage.py (Pitfall 2) |
| T-01-07 | Tampering | trv.py setting wrong hvac mode (auto) on TRVs | mitigate | trv.py hardcodes hvac_mode "heat"; acceptance criteria assert no "auto" string in trv.py; tests assert exactly two calls in heat-then-temperature order (INFRA-04) |
| T-01-08 | Denial of Service | Unavailable TRV causing service-call exception that aborts a control sweep | mitigate | trv.py guards on hass.states.get returning None or "unavailable" and returns early (ROOM-03); test covers both missing and unavailable states |
| T-01-09 | Elevation of Privilege | Config flow accepting injected user data | accept | Config flow accepts NO user fields (single Submit per D-04); RESEARCH Security Domain rates V5 input validation as not applicable for Phase 1 |
