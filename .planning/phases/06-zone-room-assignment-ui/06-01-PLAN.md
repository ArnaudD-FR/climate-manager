<objective>
Lay the foundation for the zone management UI by adding the 6 missing WebSocket client methods and building the reusable zone-tab component. This is the single largest piece of new code in Phase 6 and gates every subsequent plan.

Purpose: Phase 5 shipped the backend zone WS commands (create_zone, delete_zone, rename_zone, set_zone_mode, set_zone_time_program, reset_zone_time_program) but the frontend WsClient has zero zone methods. Without the typed wrappers, no other component can talk to the backend. The zone-tab component itself is the per-zone editor (UI-04) used by both the Default Zone tab and every custom zone tab — building it once with an `isDefault` switch avoids duplicating the mode picker + time-bar + assigned-rooms logic twice.

Output:
- frontend/src/ws-client.ts gains 6 zone methods plus a `ZoneConfig` import
- frontend/src/components/zone-tab.ts new file (~300 lines) — Lit component implementing D-04, D-05, D-06, D-08, D-09, D-10
</objective>
