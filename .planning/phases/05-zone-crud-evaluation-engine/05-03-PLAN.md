<objective>
Refactor coordinator.py's async_evaluate from a top-level global_mode switch to a
per-room zone-aware dispatch loop. Add the _resolve_zone_config(area_id, config) helper.
Remove the obsolete _evaluate_time_program and _evaluate_time_program_presences methods
(their per-room iteration logic is inlined into the unified loop). Add 5 new tests to
tests/test_coordinator.py covering EVAL-01..05.

Purpose: This is the architectural pivot of the v1.1 milestone. Per D-07/D-08,
global_mode becomes the Default Zone's mode rather than a system-wide override.
Without this refactor, zone CRUD (Plans 01+02) has no observable effect on TRV
behavior — the coordinator would still iterate all rooms under one global_mode
branch. After this plan, every room is independently resolved via its zone_id
and the zone's mode/schedule.

Output: Refactored coordinator.py + extended \_make_runtime_config helper + 5
new tests in tests/test_coordinator.py. Public API (async_evaluate signature,
status payload shape, status_update event name) unchanged. </objective>
