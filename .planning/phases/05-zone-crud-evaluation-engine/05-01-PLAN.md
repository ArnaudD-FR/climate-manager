<objective>
Add three WebSocket commands for the simple zone CRUD operations: create_zone, rename_zone,
set_zone_mode. These three are the smallest, lowest-risk handlers — they mutate only
runtime_config["zones"][zone_id] (or runtime_config["default_zone_name"] for the Default
Zone rename branch) and do not touch rooms. They all follow the existing factory pattern
from set_global_mode (websocket.py:221-245) without CR-01 snapshot-rollback complexity.

This plan implements 3 of the 6 separate granular WS commands per D-04 (CRUD as
independent commands rather than one set_zone_config aggregate, mirroring the
existing set_global_mode / set_time_program / set_room_config split). Plan 02
delivers the remaining three (delete_zone, set_zone_time_program,
reset_zone_time_program).

Purpose: Establish the zone CRUD WS contract incrementally. These three handlers
carry the simplest validation rules (vol schema only, no validate_daily_program
gate, no room migration). Plan 02 handles the more involved handlers
sequentially after this lands so it can reuse the imports and register-block
additions without merge friction.

Output: 3 new factory functions + 3 async_register_command calls in
websocket.py, and 5 new integration tests in tests/test_websocket.py.
</objective>
