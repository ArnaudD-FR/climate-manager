## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HA event loop → coordinator listeners | state_changed events drive calibration; untrusted only insofar as malformed/partial states |
| stored matter_mappings → coordinator dispatch | config values consumed when building to_set and resolving platforms |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Tampering | matter_mappings values pointing at non-climate / arbitrary entity_ids | mitigate | Coordinator resolves real platform at runtime via entity_reg.async_get().platform; only climate entities reach _push_safely (is_trv_entity guard / climate.set_temperature target); non-climate keys produce no service call. Payload-side prefix filtering is enforced in plan 02 WS handler. |
| T-13-02 | Denial of Service | Ghost listener accumulation from repeated _async_refresh_matter_listeners | mitigate | Cancel-all-then-clear before rebuild (D-10); safe because WS calls are serialised on the HA event loop. Covered by test_refresh_matter_listeners_cancels_old + test_unload_cancels_matter_listeners. |
| T-13-03 | Denial of Service | Calibration storm — every Matter state_changed firing a full calibrate pass | mitigate | Listener filters to current_temperature changes only (D-09) and schedules room-scoped _async_calibrate_for_room, not the full multi-room pass; old_state=None guarded. |
| T-13-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan (RESEARCH §Package Legitimacy Audit: not applicable — code-only). |
