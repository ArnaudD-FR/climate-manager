<objective>
Extend the existing `_make_ws_set_person_config` handler in `websocket.py` so
that when the incoming config sets `schedule_type == "even_odd"` and the stored
person has no `schedule_even` key yet, the handler deep-copies the person's
existing `schedule` into both `schedule_even` and `schedule_odd` (as two
independent copies) before the sparse-merge. Reverting to `single` and any
re-switch must leave already-stored week schedules untouched.

Purpose: This is the conversion/seeding behaviour for even/odd scheduling
(SCHED-05) plus the WS write path that accepts the new schedule_type field
(SCHED-01), and the revert-preservation guarantee (SCHED-06). No new WS command
is added — the existing set_person_config sparse-merge is extended (D-03).
Output: Modified `websocket.py`, new test cases in `tests/test_websocket.py`.
