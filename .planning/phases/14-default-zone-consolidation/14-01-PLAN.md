<objective>
Reshape the persisted config schema so the Default Zone is a single
first-class `default_zone: ZoneConfig` entry, and add a lazy read-time
storage compat shim that promotes the four legacy flat keys
(`global_mode`, `global_time_program`, `default_zone_name`,
`default_zone_preheat_enabled`) into `default_zone` on load (per D-01,
D-02, D-03 — ARCH-01).

Purpose: This is the schema foundation for the entire phase. The new
shape mirrors the proven custom-zone shape under `config["zones"][uuid]`,
so every downstream consumer (coordinator, websocket, frontend) can use a
single uniform code path with no Default Zone special-casing.

Output: Updated `DEFAULT_CONFIG`, a unified day-fill + compat shim in
`storage.async_load()`, and storage tests covering both legacy and
new-format load paths.
</objective>
