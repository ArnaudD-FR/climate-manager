<objective>
Make the coordinator read the Default Zone through the same uniform code
path as custom zones: `_resolve_zone_config` reads `config["default_zone"]`
(no `global_mode` special-case), `async_evaluate` records per-zone active
periods in a new `_last_zone_periods` dict, and `_build_status_payload`
emits a `zones` dict instead of the top-level `global_mode` /
`active_period` keys (per D-04, D-05, D-06 — ARCH-01).

Purpose: The evaluation engine and status payload are the backend's read
side of the new schema. Removing the special-case here is the core
ARCH-01 deliverable; the new `zones` payload is the contract the websocket
layer (plan 03) and frontend (plan 04) consume.

Output: Updated coordinator zone resolution, `_last_zone_periods`
tracking, redesigned `_build_status_payload`, and coordinator tests built
on a `default_zone`-shaped `_make_runtime_config`.
</objective>
