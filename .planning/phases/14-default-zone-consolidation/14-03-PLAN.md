Collapse the Default Zone WebSocket command surface onto the same handlers
custom zones use: remove `set_global_mode` and `reset_time_program`; extend
`set_zone_mode` and `reset_zone_time_program` to accept `zone_id="default"`;
rename `reset_room_to_global_program` → `reset_room_to_default_zone_program`;
point `set_zone_preheat`, `rename_zone`, and `create_zone` at the new
`default_zone` sub-keys; and make `ws_get_status` delegate to
`coordinator._build_status_payload()` (per D-07, D-08, D-09, D-10, D-11 —
ARCH-01).

Purpose: This is the backend write side and the stable WS contract that the
frontend (plan 04) consumes. It eliminates two commands and the
ws_get_status sensor-read duplication, finishing ARCH-01 success criterion 4
(unified code path for Default Zone and custom zones).

Output: Updated websocket command registry and handlers, and websocket tests
covering the extended default-zone commands, the removed commands, and the
delegated status payload.
</objective>
