## Default Zone Identity

| Option                            | Description                                               | Selected |
| --------------------------------- | --------------------------------------------------------- | -------- |
| Sentinel ID ("default")           | Fixed ID, coordinator checks zone_id == "default"         |          |
| is_default flag                   | Boolean flag on zone object, one zone always has it true  |          |
| First entry convention            | Default Zone = always first in dict                       |          |
| No storage entry (null = default) | zone_id absent/null means Default Zone; not stored at all | ✓        |

**User's choice:** Default Zone has no storage entry. `zone_id` absent or null =
belongs to Default Zone. Keeps v1.0 storage fully compatible.

**Notes:** User clarified this is a UI concept over existing global_mode +
global_time_program. Default Zone name stored as new top-level field
`default_zone_name`. Default Zone mode/schedule ARE the existing global_mode and
global_time_program fields — no new fields needed.

## STORAGE_VERSION Bump Safety

| Option                            | Description                                               | Selected |
| --------------------------------- | --------------------------------------------------------- | -------- |
| No bump — purely additive         | Sparse-merge handles new keys; STORAGE_VERSION stays at 2 | ✓        |
| Bump to 3 with migration callback | Explicit migration func registered in Store constructor   |          |

**User's choice:** No bump. All changes are additive — existing sparse-merge
loading handles new DEFAULT_CONFIG keys automatically.

**Notes:** Bumping would risk HA discarding v1.0 data if migration callback
isn't wired correctly. Since zone_id = null (absence), no existing room data
needs migration.

## Room zone_id Storage Convention

| Option                              | Description                                    | Selected |
| ----------------------------------- | ---------------------------------------------- | -------- |
| Not stored (absence = Default Zone) | Consistent with sparse D-11 model              | ✓        |
| Stored explicitly as null           | Every touched room has zone_id: null written   |          |
| zone_id on room (foreign key)       | room_config[room_id]['zone_id'] = uuid         | ✓        |
| room_ids on zone                    | zone stores list of room IDs                   |          |
| UUID zone IDs                       | Backend-generated, stable, collision-free      | ✓        |
| User-provided slug                  | Derived from name, requires collision handling |          |

**User's choice:** `zone_id` stored on room entry (foreign key). Default Zone
rooms have no `zone_id` stored (sparse). Custom zone IDs are UUIDs generated by
the backend.
