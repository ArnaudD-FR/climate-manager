  <how-to-verify>
    Open the Climate Manager panel in Home Assistant and expand a person card.
    Verify each behavior (from VALIDATION.md "Manual-Only Verifications"):

    1. Set Presence mode to "Scheduled" → the "Schedule type" select is
       visible. Set mode to "HA home tracking" / "Force Present" /
       "Force Absent" → the schedule-type select disappears. (D-01)
    2. With Schedule type = "Single week" → no Even/Odd buttons are rendered.
       (D-08)
    3. Set Schedule type = "Even / Odd weeks" → `[Even]` and `[Odd]` buttons
       appear above the time-bar, in order: section label → select →
       switcher → time-bar → reset button. (D-06, D-07)
    4. On a freshly expanded even/odd card, the active tab matches the
       current ISO week parity (today 2026-05-29 is ISO week 22 → "Even" tab
       active). (D-09)
    5. Click "Even", drag a time segment, then reload the panel → the change
       is on schedule_even and schedule_odd is unchanged. (D-11)
    6. Click "Odd", drag a segment, reload → schedule_odd changed,
       schedule_even unchanged. (D-12)
    7. Edit Even, switch to Odd, edit Odd, switch back to Even → the Even
       edits are still there (other week retained on tab switch). (D-13)
    8. Make edits on both weeks, reload the whole panel → both weeks' edits
       persist. (success criterion 4)
    9. In even/odd mode the reset button reads "Reset Even week to default"
       on the Even tab and "Reset Odd week to default" on the Odd tab; click
       it on Even → only the Even week resets, Odd is untouched. (D-15)
    10. Switch back to "Single week" → reset button reads "Reset to default"
        and the single time-bar (schedule) is shown unchanged. (D-05, D-14)

    Optional: confirm the current-week hint (if added) shows ISO week 22.
