<objective>
Add the pure ISO-week-parity helpers `getISOWeekNumber(date)` and
`getWeekParity(date)` as module-level exports in `person-card.ts`, with a
`node --test` test file proving they match the backend parity logic
(`date.isocalendar().week % 2 == 0` → "even"). These helpers are the
foundation the UI wiring (Plan 02) uses to pick the default active week.

Purpose: The frontend must select the same week (even/odd) the backend
evaluates so the user always edits the schedule that will actually apply
during the current ISO week. A parity mismatch silently shows the wrong
week. Goal-backward, this pure function is the highest-risk correctness
point in the phase (Research Assumption A1), so it gets a dedicated TDD
plan with a real automated test.

Output: Two exported functions in `person-card.ts` and one passing test
file `week-parity.test.ts`.
