## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Panel (browser) → backend WS | Schedule-type/schedule changes cross via the existing authenticated `setPersonConfig` WebSocket command. No NEW boundary is introduced — this command and its auth predate this phase. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-02 | Tampering | schedule_type value sent to setPersonConfig | mitigate | The select offers only the two literal values "single"/"even_odd"; the TypeScript union `"single" \| "even_odd"` constrains _onScheduleTypeChange. Backend (Phase 7) treats any non-"even_odd" value as "single", so an out-of-range value degrades safely to default behavior. |
| T-08-03 | Information Disclosure | render template / week switcher | accept | UI renders only the person's own already-loaded config; no additional data is fetched or exposed. WebSocket auth is handled by HA (existing). |
| T-08-04 | Denial of Service | _onSchedulePeriodsChanged sending wrong/both week fields | mitigate | Handler sends exactly one computed `field` per edit (schedule_even XOR schedule_odd XOR schedule); the other week is never included, preventing accidental cross-week overwrite. Acceptance criteria assert the single-field payload. |
| T-08-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan (RESEARCH.md "No New Packages"). `make build` uses the already-installed Vite/TypeScript toolchain. Package Legitimacy Gate not applicable. |
</threat_model>
