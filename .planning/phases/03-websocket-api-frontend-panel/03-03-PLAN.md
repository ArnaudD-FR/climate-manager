<interfaces>
<!-- Contracts the panel consumes. From Wave 2 + RESEARCH Patterns 5/6 + UI-SPEC. -->

WS commands (panel → backend, all prefixed climate_manager/):
- get_status {} -> {global_mode, active_period, present_persons, rooms_status[]}
- get_config {} -> full merged config dict
- set_global_mode {mode}; set_period_temperatures {temperatures}
- set_time_program {program:{mon:[],...,sun:[]}}
- set_room_config {room_id, config}; set_person_config {person_id, config}
- subscribe_status {} -> push events with the get_status payload shape

Per-day program shape (D-01): {"mon":[{"start":"HH:MM","mode":"<period>"}],...,"sun":[...]}
Presence schedule shape: {"mon":[{"start":"HH:MM","state":"present"|"absent"}],...}
Period mode colors (D-03, fixed hex — NOT theme vars):
  frost_protection #1565C0, reduced #64B5F6, normal #F57C00, comfort #D32F2F,
  present #388E3C, absent #9E9E9E

HA panel contract (RESEARCH Frontend Architecture):
- HA passes properties: hass (HomeAssistant), narrow (bool), panel (config)
- hass.connection.sendMessagePromise({type, ...}) -> Promise<result>
- hass.connection.subscribeMessage(cb, {type}) -> Promise<unsubscribe fn>
- customElements.define name MUST equal "climate-manager-panel" (PANEL_COMPONENT_NAME in __init__.py)

Vite single-file requirement (RESEARCH Pattern 6 / Pitfall 8):
- build.lib.entry = src/main.ts, formats ["es"], fileName "panel"
- build.outDir = ../custom_components/climate_manager/www, emptyOutDir true
- rollupOptions.output.inlineDynamicImports true, entryFileNames "panel.js"
- cssCodeSplit false  (CSS inlined into panel.js)
- npm deps: lit@3.x, typescript@5.x+, vite@5.x+, home-assistant-js-websocket@latest

Time-bar interaction contract (UI-SPEC "Time Bar Component" + D-04..D-09):
- props: days (7-element array of period arrays), mode "schedule"|"presence"
- click empty bar -> snap to nearest 15 min -> mode popup -> apply to new segment
- click block -> popup: time range + Change mode + Delete (delete merges into left neighbor)
- drag boundary -> tooltip HH:MM, 15-min snap, emit only on pointerup
- copy/paste per day row via panel-local clipboard
- leftmost block always starts at 00:00; bar always fully covered, min 15-min segment
- emits CustomEvent "periods-changed" {dayIndex, periods} on interaction END only
</interfaces>
