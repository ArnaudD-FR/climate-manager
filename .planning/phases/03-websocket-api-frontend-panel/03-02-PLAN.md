<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| panel (browser) → WS command handlers | Authenticated HA WebSocket connection; payload fields are untrusted user input |
| WS payload → store + coordinator | Mutations are persisted and pushed to physical TRVs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Tampering | Malformed payload (invalid mode, non-numeric temperature) | mitigate | `vol.In(VALID_MODES)` on set_global_mode; `vol.Coerce(float)` on every temperature; `vol.Required` type checks on every command schema — invalid messages rejected before the handler runs (ASVS V5) |
| T-03-05 | Tampering | set_time_program writes a partial/invalid per-day program | mitigate | `validate_daily_program` gate; on failure `connection.send_error` and `return` BEFORE save/evaluate — invalid program never persisted |
| T-03-06 | Elevation of Privilege | Unauthenticated config mutation | mitigate | HA WebSocket auth gate — unauthenticated connections cannot dispatch any registered command (transferred to HA core auth) |
| T-03-07 | Tampering | get_status trusts a panel-supplied / stale sensor entity id | mitigate | get_status never accepts an entity id from the payload; it reads only configured sensor ids and guards every `hass.states.get(...)` for None before reading state |
| T-03-08 | Information Disclosure | runtime_config exposed to non-admin users | accept | `require_admin=False` is the locked design (RESEARCH Security Domain V4) — all HA users may configure the integration; config contains no secrets |
| T-03-09 | Denial of Service | Sparse-storage corruption bloating stored config on every save | mitigate | Write handlers mutate only the targeted sub-key of runtime_config; DEFAULT_CONFIG is never imported or saved by websocket.py (Pitfall 5) |
</threat_model>
