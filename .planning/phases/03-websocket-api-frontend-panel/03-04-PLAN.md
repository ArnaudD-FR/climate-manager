| frontend → set_room_config WS handler | Untrusted room_id and config dict cross into backend; existing handler already does sparse merge into rooms[room_id] |
| stored runtime_config → coordinator | Stored data is authoritative — coordinator reads room_mode without re-validation each tick |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04-01 | Tampering | set_room_config payload accepts arbitrary room_mode string | mitigate | coordinator branches treat unknown room_mode as "global" (defensive default); no crash on bad value. No new WS command needed; vol schema for set_room_config already requires {room_id: str, config: dict} |
| T-03-04-02 | Information Disclosure | room_mode is non-sensitive config | accept | room_mode value is purely a behavioral switch; no PII |
| T-03-04-03 | Denial of Service | A malicious payload setting room_mode=frost_protection for all rooms would not lower temperature below frost protection setpoint (7°C default) | accept | matches GLOBAL-02 MODE_OFF semantic; user can recover via WS by setting room_mode back to "global" |
