indices 0..6. Convert in both directions at the tab boundary.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input fields → ws-client → backend | All tab field values are user input persisted to the integration store |
| live status push → DOM render | subscribe_status payload is rendered into the status strip |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-14 | Tampering | Free-text sensor/TRV entity id typed in a field | mitigate | Field value is sent only via the typed set_room_config command; backend get_status guards `hass.states.get(...)` for None (Wave 2 T-03-07) so an invalid id renders no value, not a crash; UI-SPEC shows "Entity not found" helper on blur |
| T-03-15 | Tampering | Per-room/person program saved with missing day keys | mitigate | Tab converts a full 7-key DailyProgram before save; the global program path additionally passes server-side validate_daily_program (Wave 2 T-03-05) |
| T-03-16 | Information Disclosure | Status push rendered as raw HTML | mitigate | Lit template binding escapes interpolated values by default (no unsafeHTML used for status/person/period names) |
| T-03-17 | Repudiation | User unsure whether a change saved | mitigate | Every successful save shows a "Saved" toast; failures show a persistent "Save failed — retrying..." toast (D-10) |
</threat_model>
