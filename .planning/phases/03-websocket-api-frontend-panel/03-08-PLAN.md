<objective>
Implement D-21 backend: rename the two `force` presence-mode wire values and add the new `ha` mode that derives presence from HA's own `person.*` entity state. This change is the backend pre-requisite for the frontend mode selector (Plan 03-09) and must land first so the frontend's WS payloads match what the backend stores and the coordinator evaluates.

Purpose: D-21 changes presence semantics: `"present"` and `"absent"` become
`"force_present"` and `"force_absent"` (clearer intent), and a new `"ha"` mode
delegates presence detection to HA's `person.*` entity (`state == "home"`). This
unlocks the panel's 4-option mode selector and the HA-driven presence dot
indicator on person cards.

Output: const.py wire values updated; resolve_presence() updated to match;
coordinator HA-mode branch added in `_compute_present_persons`; storage
migration for existing user data; test coverage for migration, HA-mode branch,
and absence of regressions across schedule/coordinator/storage suites.
</objective>
