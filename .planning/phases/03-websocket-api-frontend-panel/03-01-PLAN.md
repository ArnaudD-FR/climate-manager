<objective>
Refactor the Phase 2 backend from the `weekday_groups` time-program schema to the
per-day schema (`{"mon": [...], "tue": [...], ..., "sun": [...]}`) locked by
CONTEXT.md D-01. This is a mandatory pre-condition gap-closure plan: every Phase 3
WebSocket command and the entire frontend panel are designed against the per-day
schema exclusively. No Phase 3 panel/API work can begin until this completes and
the test suite is green against the new schema.

Purpose: D-01 requires a per-day schema for global time programs, per-room
overrides, and person presence schedules. The Phase 2 backend uses a
`weekday_groups` list-of-groups structure that is incompatible with the panel's
7-bar-per-day editor and the WebSocket command payloads.

Output: `const.py`, `schedule.py`, `coordinator.py`, and the three affected test
files all converted to the per-day schema, full `pytest tests/` green.
</objective>
