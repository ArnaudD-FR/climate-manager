Wire calendar lookups into the coordinator: add a per-cycle `_calendar_cache`,
an async `_prefetch_calendars()` that issues one `calendar.get_events` call per
unique entity using `asyncio.gather()`, reset+prefetch at the top of
`async_evaluate()`, and route calendar-mode persons and calendar period states
through the cache in both `_compute_present_persons()` and
`_apply_presence_overrides()`.

fetch per entity, silent fallback). This is the async glue that feeds Plan 01's
pure helpers — keeping the heavy lifting out of schedule.py.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| coordinator → calendar service | get_events crosses into the HA service bus (already authenticated); entity_id is config-supplied |
| stored config → service call | `calendar_config.entity_id` from persisted config is used as a service target |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-03 | Information Disclosure / Misuse | `entity_id` used as get_events target | mitigate | `_prefetch_calendars` skips any entity_id not starting with "calendar." (ASVS V5) — prevents calling unintended services |
| T-11-04 | Denial of Service | calendar entity error spamming logs every cycle | mitigate | HomeAssistantError caught, single WARNING per failing entity per cycle, empty-list fallback (D-04); no retry storm |
| T-11-05 | Tampering | malformed get_events response shape | mitigate | `(result or {}).get(eid, {}).get("events", [])` — defensive access, never raises KeyError (Landmine 2) |
| T-11-SC | Tampering | npm/pip/cargo installs | accept | No package installs (D-02, manifest requirements stays []); no [ASSUMED]/[SUS] packages |
</threat_model>
