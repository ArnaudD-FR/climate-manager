## Root Cause

`discover_rooms` calls `entity_reg.entities.get_entries_for_area_id(area.id)`
which only returns entities where `entity.area_id == area_id`. Most HA users
assign areas to **devices** (not individual entities): the entity's `area_id` is
`None` and it inherits from its device. Result: discovery finds no climate
entities → rooms_status is empty → Rooms tab shows "No rooms discovered".

## Fix

Update `custom_components/climate_manager/discovery.py`:

1. Import `device_registry as dr`
2. Build a two-pass index over entity_reg:
   - Pass 1: direct entity → area assignment (existing logic, restructured)
   - Pass 2: device → area assignment (entity.area_id is None, device.area_id ==
     area_id)
3. Merge both passes, deduplicate by entity_id
