<objective>
Replace the Matter pairing `<select>` UI in the room card with a
drag-and-drop entity grouping canvas. Each Tado X entity in the room
gets a group box; unassigned Matter entities appear as draggable chips
below. Dropping a chip onto a group box calls `ws.setMatterMapping`.

Purpose: The select UI only supports one-to-one mapping and is clunky
for rooms with multiple Tado X valves. The DnD canvas makes multi-
entity mapping visual and immediate.

Output: Updated `room-card.ts` with `_renderClimateSection()` replacing
`_renderMatterPairingSection()`; new grouping styles in `shared-styles.ts`.
</objective>

    **Add `_renderClimateSection()`.** Logic:

    1. Read `tadoXEntities = this.panelConfig?.tado_x_entities ?? []`
    2. Read `roomEntityIds = this.roomStatus?.entity_ids ?? []`
    3. Derive `roomTadoXIds` = intersection of `roomEntityIds` and
       `tadoXEntities`
    4. If `roomTadoXIds.length === 0` AND
       `(this.panelConfig?.matter_entities ?? []).length === 0`:
       — fall through to the existing TRV chip display (reuse
       `_renderTrvSection()` output as-is)
    5. Otherwise render the DnD canvas:

       **Section header:**
       ```html
       <div class="section-label">Real-time calibration</div>
       <p class="schedule-hint">
         Drag Matter entities onto a Tado X valve to pair them for
         sub-minute calibration.
       </p>
       ```

       **Group boxes** (one per `tadoId` in `roomTadoXIds`):
       - Outer `<div class="group-box">` with:
         - `@dragover=${this._onGroupDragOver}`
         - `@drop=${(e) => this._onMatterDropOnGroup(tadoId, e)}`
         - `@dragenter=${(e) => (e.currentTarget as HTMLElement)
             .classList.add('drag-over')}`
         - `@dragleave=${(e) => (e.currentTarget as HTMLElement)
             .classList.remove('drag-over')}`
       - Header label: friendly name of `tadoId` from
         `this.hass?.states[tadoId]?.attributes?.friendly_name ?? tadoId`
         in a `<div style="font-size:13px;font-weight:600;
         margin-bottom:6px;">`
       - Chip row: `mapped = this.panelConfig?.matter_mappings?.[tadoId]
         ?? []`; for each `matterId` in `mapped` render a `.matter-chip`:
         ```html
         <span class="matter-chip">
           ${this.hass?.states[matterId]?.attributes?.friendly_name
             ?? matterId}
           <button @click=${() =>
             this._onUnmapMatterEntity(tadoId, matterId)}>×</button>
         </span>
         ```

       **Orphan chips row:**
       - Compute `allMapped`: flatten all value arrays of
         `this.panelConfig?.matter_mappings ?? {}`
       - `orphans = (this.panelConfig?.matter_entities ?? [])
           .filter(id => !allMapped.includes(id))`
       - If `orphans.length > 0`:
         ```html
         <div class="section-label" style="margin-top:12px;">
           Unassigned Matter entities
         </div>
         <div class="orphan-chips">
           ${orphans.map(id => html`
             <span
               class="matter-chip"
               draggable="true"
               @dragstart=${(e) => this._onMatterDragStart(id, e)}
             >
               ${this.hass?.states[id]?.attributes?.friendly_name ?? id}
             </span>
           `)}
         </div>
         ```

    **Add drag handlers** (four private methods, exact signatures per spec):

    ```
    private _onMatterDragStart(matterEntityId: string, e: DragEvent): void
    private _onGroupDragOver(e: DragEvent): void
    private async _onMatterDropOnGroup(
      tadoEntityId: string, e: DragEvent): Promise<void>
    private async _onUnmapMatterEntity(
      tadoEntityId: string, matterEntityId: string): Promise<void>
    ```

    Implementations are verbatim from the task spec (see ## What to build

    `this.ws.setMatterMapping(tadoEntityId, remaining)` then
    `this.panel.reloadConfig()`. Neither shows a toast — silent save is
