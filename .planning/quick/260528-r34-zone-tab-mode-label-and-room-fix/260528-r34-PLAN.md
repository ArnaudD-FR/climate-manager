# Quick Task 260528-r34: Zone tab mode label + assigned rooms bug fix

## Task 1: Rename "Zone mode" → "Mode" with section-label style

In zone-tab.ts render(), change:
  `<label class="select-label">Zone mode</label>`
→ `<div class="section-label">Mode</div>`

The section-label CSS (already defined in zone-tab.ts) gives uppercase, bold, small font —
matching "Assigned rooms" label style.

## Task 2: Fix missing rooms in Default Zone (and custom zone) assigned rooms list

Root cause: `_getAssignedRoomIds()` and `_getUnassignedRoomItems()` iterate over
`config.rooms` — but rooms with all-default settings (no custom time program, no zone
assignment) have no entry in `config.rooms`. Those rooms are invisible to zone-tab even
though they belong to Default Zone.

Fix: Add `status: StatusPayload | null` prop to zone-tab. Use
`status.rooms_status.filter(has_trv !== false)` as the complete room universe for both
`_getAssignedRoomIds()` and `_getUnassignedRoomItems()`. Fall back to `config.rooms` keys
when status is not yet available.

Also update `_getRoomName()` to prefer the status name (from HA area registry, correctly
formatted) over hass.areas, for consistency with rooms-tab.

Pass `.status=${this._status}` to zone-tab in main.ts (both default and custom zone renders).
