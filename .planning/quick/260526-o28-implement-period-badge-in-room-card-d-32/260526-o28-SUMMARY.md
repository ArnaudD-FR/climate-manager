# Phase quick Plan 260526-o28: Period Badge in Room Card Header (D-32) Summary

Added a period badge to row 1 of the room card header and trimmed the status line from 4 items to 3 by removing the clock-outline period item.

## What Changed

### `frontend/src/components/room-card.ts`

**Lines added:** +43  **Lines removed:** -23  **Net:** +20

**New helper `_renderPeriodBadge()` (lines 444-478):**
- Returns empty TemplateResult when `resolvedMode === "frost_protection"` (mode badge already conveys state)
- Returns gray "Off" badge (`--secondary-background-color` / `--secondary-text-color`) when `globalMode === "off"`
- Returns empty TemplateResult when `active_period` is null/undefined
- Otherwise returns a colored pill using `PERIOD_COLORS[period]` as background, white text, content `"${PERIOD_DISPLAY_NAMES[period]} · ${temp}°C"` (falls back to just the name if `period_temperatures[period]` is missing)
- Reuses existing `.program-badge` CSS class with inline `style` for color (consistent with frost mode badge pattern)

**Row 1 `card-header-top` change (line 608):**
- Inserted `${this._renderPeriodBadge()}` between the room name span and the existing mode badge span
- Order is now: `[room name] [period badge] [mode badge]`
- Mode badge (frost/custom/global) unchanged

- Removed the `periodDisplay` computation block (the `globalMode === "off"` branch + period/temp lookup)
- Removed the `<span class="status-item">` with `mdi:clock-outline` that rendered the period display
- Row 2 now has exactly 3 status items: `mdi:thermometer` (temp) · `mdi:water-percent` (humidity) · `mdi:account-group` (persons)
