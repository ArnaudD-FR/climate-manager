# Quick Task 260603-uo0: Matter→Tado X Drag-and-Drop Pairing UI

## Goal

Replace the old Matter pairing select-based UI in `room-card` with a
drag-and-drop tree. Add a panel.js cache-bust so deployed JS is always
fresh in the browser.

## Tasks

### Task 1 — DnD pairing UI in room-card.ts

**Files:** `frontend/src/components/room-card.ts`

Rewrite `_renderClimateSection()` and add all supporting state/handlers:

- `@state() _dragActive` / `_dragType` — track active drag type
- `_entityName(id)` — friendly name helper
- `_startDrag` / `_onDragEnd` — drag lifecycle
- `_onMatterDragOver` / `_onAnyDragOver` — conditional dragover prevention
- `_onMatterDragEnter` / `_onMatterDragLeave` — flicker-free group highlight
- `_parseDragPayload` — safe JSON drag data parse
- `_onDropOnTado(tadoId)` — assign Matter entity to Tado X group
- `_onDropOnUnassign` — unmap Matter from group, or dissolve Tado X group

Render logic:

- `canPair = roomTadoXIds.length > 0 && matterEntities.length > 0`
- L1 (`.climate-tree`): Tado X groups + independent entities, sorted by
  friendly name
- L2 (`.matter-children`): Matter children under each Tado X group, sorted
  by friendly name
- Unassign zone rendered on the right only while `_dragActive`
- All chips use `mdi:thermometer`; draggable chips get `.chip-draggable`

### Task 2 — groupDndStyles in shared-styles.ts

**Files:** `frontend/src/shared-styles.ts`

New `groupDndStyles` export:

- `.climate-pair` — flex row (tree + unassign zone during drag)
- `.climate-tree` — flex column, always present
- `.tado-group` + `.tado-group.drag-over` — group container with drop
  highlight
- `.matter-children` — flex column, indented with left border
- `.chip-draggable`, `.chip-temp`, `.chip.drag-over`
- `.unassign-drop-zone` + `.unassign-drop-zone.drag-over`

### Task 3 — panel.js cache-bust in __init__.py

**Files:** `custom_components/climate_manager/__init__.py`

- Add `import json` at module top
- Read `manifest.json` version at `async_setup_entry` time
- Append `?v=<version>` to `module_url` so every release forces a fresh
  browser download, preventing stale-cache bugs after `make deploy`
