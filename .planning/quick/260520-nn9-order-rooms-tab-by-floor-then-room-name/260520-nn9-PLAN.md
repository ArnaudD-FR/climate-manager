# Quick Task 260520-nn9: Order Rooms Tab by Floor then Room Name

## Goal

Replace the current custom-program-first ordering in the Rooms tab with floor → room name ordering, matching the HA climate panel. Show floor names as section headers between floor groups.

## Context

Decisions captured in .planning/phases/03-websocket-api-frontend-panel/03-CONTEXT.md (D-14a, D-14b):
- Order: floor.level ascending → room name alphabetical within floor
- Floorless rooms: at end, alphabetical, no section header
- Floor section headers: yes, show floor name (e.g. "Ground floor")
- Data source: hass.areas[area_id].floor_id + hass.floors[floor_id].{level, name}
- Frontend only — no backend changes

## Tasks

### Task 1: Extend Hass type with areas and floors

**File:** `frontend/src/types.ts`

Add to the `Hass` interface:
```typescript
areas: Record<string, { area_id: string; name: string; floor_id: string | null }>;
floors: Record<string, { floor_id: string; name: string; level: number }>;
```

### Task 2: Rewrite rooms-tab sort logic and render floor section headers

**File:** `frontend/src/components/rooms-tab.ts`

Replace the current `sortedIds` sort (custom-program first) with floor-based grouping:

1. Build a map of floor_id → sorted room IDs (alphabetical by name within floor)
2. Sort floors by level ascending
3. Collect floorless room IDs alphabetically
4. Render: for each floor group → section header + room cards; then floorless rooms (no header)

Add CSS for `.floor-header` section label.

### Task 3: Build

Run `make build` to compile TypeScript to panel.js. Verify no TypeScript errors.
