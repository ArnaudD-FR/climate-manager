# Quick Task 260520-s9s: Remove sensors section from room card and fix backend sensor priority chain

## Goal

Remove the manual sensor picker UI that was added in 260520-t4u (wrong approach
— sensor configuration belongs in HA area settings, not Climate Manager). Update
the backend priority chain in both `websocket.py` and `coordinator.py` to use
`AreaEntry.temperature_entity_id` / `humidity_entity_id` (HA 2026.5+) as primary
source, falling back to auto-discovered sensors, then TRV `current_temperature`,
then omit key (frontend shows "—").

## Tasks

### Task 1: Remove sensors section from room-card.ts and RoomConfig types

**Files:**

- `frontend/src/components/room-card.ts`
- `frontend/src/types.ts`

**Action:**

1. Remove CSS blocks: `.sensor-section`, `.sensor-row`, `.sensor-row-label`,
   `.sensor-input` (and focus variant)
2. Remove `_onSensorBlur()` method
3. Remove `_renderSensorsSection()` method
4. Remove `${this._renderSensorsSection()}` call from `render()`
5. Remove `temperature_sensor` and `humidity_sensor` fields from `RoomConfig`
   interface in types.ts

### Task 2: Update backend sensor priority chain

**Files:**

- `custom_components/climate_manager/websocket.py` (ws_get_status)
- `custom_components/climate_manager/coordinator.py` (\_build_status_payload)

**Action:** In both locations, replace:

```python
temp_sensor = room_cfg.get("temperature_sensor") or auto.get("temperature")
```

with:

```python
# Primary: HA area registry designated sensors (HA 2026.5+)
temp_sensor = getattr(_area, 'temperature_entity_id', None)
humidity_sensor = getattr(_area, 'humidity_entity_id', None)
# Fallback: auto-discovered area sensors
if not temp_sensor:
    temp_sensor = auto.get("temperature")
if not humidity_sensor:
    humidity_sensor = auto.get("humidity")
```

Add `unavailable`/`unknown` state guards when reading sensor state values.
Remove `room_cfg.get("humidity_sensor")` references.
