# Quick Task 260520-t4u: Room Sensor Pickers

## Root Cause

`RoomConfig` has `temperature_sensor` and `humidity_sensor` fields and the backend
reads them (websocket.py ws_get_status, coordinator.py _build_status_payload), but the
room card UI has no controls to set them. Users cannot point rooms to their dedicated
ambient sensors — values fall back to Tado's own TRV thermistor.

## Fix

Add a "Sensors" section to the room card expanded body, after the TRV section:
- Temperature sensor: `ha-entity-picker` filtered to `sensor` domain with `.hass` passed in
- Humidity sensor: same
- On `value-changed` event: call `ws.setRoomConfig(roomId, { temperature_sensor: value })`
  then `panel.reloadConfig()` + `panel.showToast("Saved", false)`
- Empty string / null clears the override (falls back to auto-detected sensor)

Files to modify:
- `frontend/src/components/room-card.ts`
  - Add `_renderSensorsSection()` method
  - Add `_onSensorChange(field, value)` handler
  - Insert section between TRV section and persons section in render
  - Add minimal CSS for the section label
