# Milestone v1.1: Heating Zones

**Status:** ✅ SHIPPED 2026-05-28
**Phases:** 4-6
**Total Plans:** 9

## Overview

Adds named heating zones that group rooms under independent modes and weekly
schedules. Every install always has a non-deletable Default Zone; custom zones
can be created, renamed, and deleted through the panel. The coordinator was
refactored from a global-mode switch to a per-room zone-aware dispatch loop.

## Milestone Summary

**Key Decisions:**

- D-01: Default Zone has no storage entry — absent `zone_id` = Default Zone
  membership (sparse model, no sentinel ID)
- D-02: Default Zone mode/schedule IS `global_mode`/`global_time_program` — no
  new fields for Default Zone
- D-04: STORAGE_VERSION stays at 2 — all changes purely additive
- D-07: Zone IDs are UUID strings (generated by `uuid.uuid4()` in backend)
- UI: HA 2026.x compatibility — native `<input>` and `<select>` only after
  discovering `ha-textfield`, `ha-select`, `ha-tabs` broken in latest HA
