# Requirements — Climate Manager v1.1 Heating Zones

## Overview

This milestone adds named heating zones to the Climate Manager. A zone is a
group of rooms that runs its own independent mode and weekly schedule. The
system always has a non-deletable Default Zone; all rooms belong to exactly one
zone at all times.

---

## v1.1 Requirements

### Zone Data Model

- [x] **ZONE-01**: System stores named zones with id, name, mode, and weekly
      time program
- [x] **ZONE-02**: System always has exactly one non-deletable Default Zone; all
      rooms belong to a zone at all times
- [x] **ZONE-03**: Storage schema loads from v1.0 installs cleanly — rooms
      without a zone are assigned to the Default Zone on first load (no data
      loss, no manual migration)
- [x] **ZONE-04**: A room can belong to at most one zone (enforced at save time)

### Zone CRUD

- [x] **ZONE-05**: User can create a new custom zone with a name
- [x] **ZONE-06**: User can rename any zone including the Default Zone
- [x] **ZONE-07**: User can delete a custom zone — its rooms are moved to the
      Default Zone automatically; the Default Zone cannot be deleted
- [x] **ZONE-08**: User can set any zone's mode (Off / Time program / Time
      program & presences)
- [x] **ZONE-09**: User can edit any zone's weekly time program (same
      weekday-group + period structure as global)

### Zone Backend Evaluation

- [x] **EVAL-01**: Zone mode=off → all rooms in the zone receive
      frost-protection temperature
- [x] **EVAL-02**: Zone mode=time_program → rooms run the zone's weekly schedule
- [x] **EVAL-03**: Zone mode=time_program_presences → rooms run zone schedule
      with presence override
- [x] **EVAL-04**: Global mode=time_program_presences → presence heating applies
      to all rooms regardless of zone mode (global presence governs)
- [x] **EVAL-05**: Per-room custom schedule overrides zone schedule (room-level
      has highest priority)

### Room Assignment

- [x] **ASSIGN-01**: User can assign rooms to a zone from within the zone tab
- [x] **ASSIGN-02**: User can assign/change a room's zone from within the room
      card
- [x] **ASSIGN-03**: Room card shows a zone badge (zone name) for every room

### Panel UI Redesign

- [x] **UI-01**: Tab bar order: Overview | Default Zone | [custom zone tabs] |
      Rooms | Persons
- [x] **UI-02**: Custom zone tabs are added and removed dynamically as zones are
      created or deleted
- [x] **UI-03**: User can create a new zone from the panel
- [x] **UI-04**: Every zone tab shows zone name (inline editable), mode picker,
      weekly time program (reusing the time-bar component), and list of assigned
      rooms
- [x] **UI-05**: Custom zone tabs have a delete button with confirmation; Default
      Zone tab does not
- [x] **UI-06**: Rooms tab shows all rooms with zone badge and zone picker on
      each room card

## Out of Scope (v1.1)

| Item                        | Reason                                              |
| --------------------------- | --------------------------------------------------- |
| Per-zone temperature setpoints | Adds data model complexity; global period temps apply to all zones |
| Zone priority ordering      | All zones independent; no ranking needed            |
| Hierarchical zones          | No use case identified                              |
| Zone-based presence         | Presence detection remains person→room              |
| "Copy zone" shortcut        | Defer until zones validated in production           |
